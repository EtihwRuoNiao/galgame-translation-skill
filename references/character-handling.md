# 字符处理详细规则

本文档详细说明 `galgame-translation-skill` 中各种字符的处理规则。  
核心原则：**所有字符（包括格式符、英数等字符）严格与原文保持一致的形态，不进行任何主动转换**。

---

## 格式提示字符处理

### 严格保留原文形态

| 字符 | 处理方式 |
|------|----------|
| `★◎` | 与原文严格一致（半角/全角均按原文） |
| `◎★` | 与原文严格一致（半角/全角均按原文） |
| `//` | 与原文严格一致（半角/全角均按原文） |
| `#0x...` | 与原文严格一致（十六进制数字保持原形态） |
| `@...` | 与原文严格一致（如 `@b1@t24@w1`） |
| `\n` | 与原文严格一致（转义字符不解释） |
| `｛...｝` | 与原文严格一致（花括号注音） |
| `◆◆` | 与原文严格一致（数量、全角/半角均不变） |
| `{...}` | 与原文严格一致（变量占位符） |
| `　` | **全角空格**（U+3000），与原文严格一致：原文目标行开头有全角缩进则译文必须保留相同数量的全角空格；行中出现的全角空格同样保留，不得转为半角空格或移除 |

### 示例

```
原文：★◎  001  ◎★//Hello
译文：★◎  001  ◎★你好        ← ★◎ ◎★ // 保持半角（假设原文是半角）

原文：★◎  001  ◎★//Hello
译文：★◎  001  ◎★你好       ← ★◎ ◎★ 保持全角（假设原文是全角，罕见情况）
```

## 数字处理规则

### 与原文形态严格一致

- 原文半角 `123` → 译文保持半角 `123`
- 原文全角 `１２３` → 译文保持全角 `１２３`
- **原则**：不主动转换数字的全角/半角状态

### 混合情况

- 中文句子中的数字：根据原文形态决定
  - 原文：`我有123个苹果` → 译文：`我有123个苹果`（半角保持）
  - 原文：`我有１２３个苹果` → 译文：`我有１２３个苹果`（全角保持）
- 数字与格式字符混合：
  - 原文：`#0x1234` → 译文：`#0x1234`（十六进制数字保持半角）
  - 原文：`#0x１２３４` → 译文：`#0x１２３４`（保持全角）

---

## 英文字母处理规则

### 与原文形态严格一致

- 原文半角 `ABC` → 译文保持半角 `ABC`
- 原文全角 `ＡＢＣ` → 译文保持全角 `ＡＢＣ`
- **原则**：不主动转换英文字母的全角/半角状态

### 专有名词中的英文

- 词典中已定义的英文译名：按词典执行（如果词典指定了形态）
- 未定义的英文：与原文形态严格一致
- 示例：
  - 原文：`D.C.` → 译文：`D.C.`（半角保持）
  - 原文：`Ｄ．Ｃ．` → 译文：`Ｄ．Ｃ．`（全角保持）

---

## 中文文本处理

### 译文中的中文

- **汉字**：全角（默认）
- **中文标点**：全角（`，。！？；：""''（）【】` 等）
- **与原文关系**：中文文本不受原文英数形态影响

### 示例

```
原文：★◎  001  ◎★//Hello World！
译文：★◎  001  ◎★你好，世界！

（★◎ ◎★ 保持半角，中文使用全角标点，英文 Hello World 已被翻译）
```

---

## 混合字符处理

### 译文与原文形态混合

示例：
```
原文：★◎  001  ◎★//我有123个苹果和ABC。
译文：★◎  001  ◎★我有123个苹果和ABC。

（★◎ ◎★ 保持半角，中文全角，数字123保持半角，英文ABC保持半角）
```

---

## 注音字符处理

### 花括号注音 `｛...｝`

- **原样保留**：不修改括号内的任何内容
- 注音文本保持原有编码和字符形态
- 示例：`｛さき／咲姫｝` → 原样保留

### 其他注音形式

- 如遇 `【...】`、`（...）` 等形式的注音，同样原样保留。

---

## 引擎格式码处理

### 常见格式码

- `@b1@t24@w1` — 按钮、时间、等待等指令
- `@...` 格式：保持原有形态（半角/全角按原文）
- 格式码前后的文本正常翻译。

### 示例

```
原文：@b1@t24@w1★◎  001  ◎★//こんにちは。
译文：@b1@t24@w1★◎  001  ◎★你好。

（格式码 @b1@t24@w1 原样保留，★◎ ◎★ 保持半角）
```

---

## 转义字符处理

### 换行符 `\n`

- **原样保留**：不解释为真正的换行
- 保持原有形态（`\` 和 `n` 的形态按原文）
- 示例：`TEXT\nMORE` → 输出 `翻译文本\nMORE`

---

## 占位符处理

### 常见占位符

- `◆◆`、`◆`、`{...}` 等
- **原样保留**：不翻译，不改变形式
- **数量一致**：原文有几个占位符，译文就输出几个（全角/半角按原文）
- **形态一致**：原文 `◆◆`（双半角）则译文 `◆◆`，原文 `◆◆`（双全角）则译文 `◆◆`

### 示例

```
原文：★◎  001  ◎★//◆◆
译文：★◎  001  ◎★◆◆

（占位符 ◆◆ 保持半角、双占位符数量一致）

原文：★◎  002  ◎★//◆
译文：★◎  002  ◎★◆

（单占位符保持一致）
```

---

## 检查清单

翻译完成后，检查：
- [ ] 格式字符（`★◎`、`◎★`、`//`、`　` 等）与原文形态严格一致
- [ ] 数字与原文形态一致（半角/全角保持不变）
- [ ] 英文字母与原文形态一致（半角/全角保持不变）
- [ ] 中文文本使用全角标点（，。！？等）
- [ ] 注音字符原样保留（`｛...｝` 不修改）
- [ ] 引擎格式码原样保留（`@...` 不拆分）
- [ ] 占位符数量与形态一致（`◆◆` 保持 `◆◆`）
- [ ] 专有名词与词典一致
- [ ] 无日文式中文表达
- [ ] 输出文件编码与原始文件一致
- [ ] 输出文件换行符与原始文件一致
