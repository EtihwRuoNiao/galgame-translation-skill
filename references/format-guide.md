# 格式分析指南

本文档详细说明 galgame-translation-skill 支持的文件格式和检测逻辑。

## 文件格式类型

### 1. 双行格式（默认）

每组条目由三行构成：

```
#0x1234                            ← 地址行（不动）
★◎  0611_001  ◎★//こんにちは。     ← 源行（不动，有 // 标记）
★◎  0611_001  ◎★こんにちは。        ← 目标行（翻译此行）
```

**特征**：
- 存在 `★◎...◎★//` 模式（源行）
- 存在对应的 `★◎...◎★` 模式（目标行，无 `//`）
- 通常三行一组，地址行 + 源行 + 目标行

**处理规则**：
- 地址行：完全不动
- 源行（有 `//`）：完全不动
- 目标行（无 `//`）：替换为中文译文，保留 `★◎ NNN ◎★` 前缀

### 2. 单行格式

只有源行，无对应目标行：

```
★◎  0611_001  ◎★//こんにちは。
★◎  0611_002  ◎★//さようなら。
```

**特征**：
- 只有 `★◎...◎★//` 源行
- 无对应的非 `//` 行

**处理规则**：
- 为每个源行生成对应的目标行
- 格式：`★◎ NNN ◎★` + 中文译文

## 格式提示字符

以下字符在输出中**一律原样保留**（含全角/半角形态）：

| 字符 | 说明 |
|------|------|
| `★◎` | 段落开始标记（保持原文半角） |
| `◎★` | 段落结束标记（保持原文半角） |
| `//` | 源语言标记（在源行末尾，保持原文半角） |
| `#0x...` | 十六进制地址行 |
| `\n` | 转义换行符 |
| `@...` | 引擎格式码（如 `@b1@t24@w1`） |
| `｛...｝` | 花括号注音 |
| `◆◆` | 占位符 |
| `{...}` | 变量占位符 |
| `　`（U+3000） | **全角空格** — 原文中的全角缩进/间距必须严格保留，不得转换为半角空格或移除 |

> **全角空格处理要点**：原文目标行如果以全角空格开头（如 `　 文本内容`），译文目标行必须保持相同数量的前置全角空格。全角空格出现在行中时同样原样保留。

### 完整模式示例

- **源行完整模式**：`★◎  0611_001  ◎★//こんにちは。`（有 `//` 标记，不动）
- **目标行完整模式**：`★◎  0611_001  ◎★こんにちは。`（无 `//`，翻译此行）
- **地址行完整模式**：`#0x1234`（十六进制地址，不动）
- **注音完整模式**：`｛さき／咲姫｝`（花括号注音，原样保留）
- **引擎码完整模式**：`@b1@t24@w1`（格式码，原样保留）

## 文件属性检测

### 编码检测
自动检测并保留原始编码：
- Shift-JIS（日文游戏常用）
- UTF-16（Unicode）
- UTF-8（Unicode）
- 其他编码（GBK、EUC-JP 等）

### 换行符检测
自动检测并保留原始换行符类型：
- CRLF (`\r\n`) — Windows 风格
- LF (`\n`) — Unix/Linux 风格

## Fallback 格式检测

当文件不匹配上述任何格式时（即无 `★◎...◎★//` 模式），启动 fallback 逻辑：

1. 读取 `assets/examples/` 下第一个范例文件
2. 读取 `input/` 下第一个待翻译文件
3. 对照分析，找出长度超过 15 个 token 的日语文本行（源行）
4. 在范例文件中找到对应的目标行（通常紧跟源行下方或按固定间隔排列）
5. 根据配对关系推导当前文件的格式结构
6. 如有多个范例文件，取多数一致的模式作为最终格式规则

### 边缘情况

#### 文件无任何格式标记
- 视为**纯文本文件**
- **逐行翻译**：每行文本直接翻译，不保留任何特殊格式
- 输出文件保持原始编码和换行符

#### 混合格式文件
- 同一文件同时存在双行格式和单行格式
- **按出现频率决定**：如果双行格式条目占比 >50%，视为双行格式；否则视为单行格式
- **混合处理**：按检测出的主要格式处理，忽略少数格式的行（或视为纯文本行）

#### 格式标记不完整
- 只有 `★◎` 没有 `◎★`：视为无效格式，按纯文本处理
- 只有 `//` 没有 `★◎`：视为注释行，不翻译
- 地址行缺失：不影响，只处理有 `★◎` 的行

#### 文件编码无法检测
- 默认使用 **UTF-8** 尝试解码
- 如果失败，尝试 **Shift-JIS**
- 如果仍失败，报错并记录到总结报告
