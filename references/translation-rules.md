# 翻译规则详解

本文档详细说明 galgame-translation-skill 的翻译规则和最佳实践。

## 核心翻译规则

### 1. 风格模仿
严格按照 `assets/examples/` 中的范例文件处理：
- **语气**：口语化 vs 书面语
- **敬语处理**：如何翻译です/ます体、である体
- **角色口吻**：不同角色的说话风格差异
- **句式习惯**：长句拆分、语序调整

### 2. 词典优先原则

词典文件位于 `assets/dictionary/`，CSV 格式，使用 `==== TYPE ====` 分区：

#### 强制区块（必须使用指定译名）
- `==== PERSON ====` — 人物名称
- `==== PLACE ====` — 地点名称
- `==== TITLE ====` — 头衔、称谓

示例：
```csv
==== PERSON ====
るる姉,露露姐,
姫乃,姬乃,注意"姫/姬"字的中日文字形不一样
```

#### 参考区块（优先参考，可酌情调整）
- `==== TERM ====` — 术语、专有名词

示例：
```csv
==== TERM ====
ダ・カーポ,D.C.（返始记号；返始咏叹调）,
枯れない桜,永不枯萎的樱花（树），不枯樱,
```

### 3. 自然度要求
- 译文符合中文表达习惯
- 避免输出日文式中文
- 使用自然的中文语序和搭配
- 适当使用中文习惯用语

### 4. 一致性原则
- 同一人物名/地名/术语在全文中译名一致
- 参考词典中的指定译名
- 无词典条目时，首次出现确定译名后全文保持一致

### 5. 一致性例外
连续的对话台词中重复出现的人名，应当酌情：
- 用第二人称（你、您）代替
- 用第三人称（他、她）代替
- 用称谓（老师、前辈）代替
- 用昵称代替

### 6. 格式完整性
所有格式标记原样保留 → 详见 `references/character-handling.md` → 格式提示字符处理

### 7. 全角/半角规则
- **译文中的中文文本及标点**：一律使用**全角字符**（，。！？等）
- **格式提示字符**：按原文**半角保留**，不做转换：
  - `★◎`、`◎★`、`//`、`#0x`、`@...`、`\n`、变量占位符等
- **全角空格**（`　`，U+3000）：严格按原文保留，**不**属于上述半角格式符，译文中行首缩进和行中间距均与原文一致

## 翻译流程

1. **加载词典** → 读取所有 CSV 词典文件，建立译名映射
2. **分析范例** → 读取示例文件，提取风格特征
3. **检测格式** → 确定文件是双行格式还是单行格式
4. **逐行翻译** → 保留所有格式字符，仅翻译日文文本部分
5. **输出文件** → 使用原始编码和换行符，写入新建的输出文件夹

## 特殊场景处理

### 占位符处理
原文是占位符（如 `◆◆`、`◆` 等）时：
- **原样保留**：译文对应位置输出**完全相同**的占位符（包括数量和形式）
- **形式一致**：原文是 `◆◆`（双占位符）则输出 `◆◆`，原文是 `◆`（单占位符）则输出 `◆`
- **不翻译占位符**：占位符内容不参与翻译
- **保留周围格式**：占位符前后的格式字符（`★◎`、`◎★`、`//` 等）原样保留

**示例**：
```
原文：★◎  001  ◎★//◆◆
译文：★◎  001  ◎★◆◆          ← 占位符完全相同

原文：★◎  002  ◎★//◆
译文：★◎  002  ◎★◆            ← 单占位符保持一致
```

### 注音处理
遇到 `｛さき／咲姫｝` 格式：
- 保留整个注音块原样不动
- 翻译周围的日文文本
- 注音内容不参与翻译

### 引擎格式码处理
遇到 `@b1@t24@w1` 等格式码：
- 原样保留，不修改
- 格式码前后的文本正常翻译
- 确保格式码不被拆分
